# CS 主队偏好测试

一个轻量的静态网页测试项目，用来判断做题者更接近哪支主流 CS 战队的风格。

## 项目简介

这个测试会通过更精简的主问题与动态加试题，结合多维度权重，给出：

- 你的主队结果
- 主队接近度
- 风格倾向
- 结果置信度
- 作答稳定度
- 其他接近主队

整个项目基于纯前端实现，不依赖打包工具，直接用本地静态服务器即可运行。

## 功能特点

- 10 道核心主测试题
- 题目顺序随机
- 选项顺序随机
- 开始欢迎页
- 自适应加试题机制
- 多维度加权匹配
- 接近结果自动细分
- 适合继续扩展战队和题库

## 当前判定维度

目前测试会综合以下维度判断：

1. 纪律性 / 个人英雄主义
2. AWP 偏好 / 步枪偏好
3. 冒险突破 / 稳扎稳打
4. 默认控场 / 非常规打法
5. 战术体系 / 纯火力压制
6. 残局处理 / 滚雪球压制
7. 经济运营 / 强起赌博

## 项目结构

- index.html：页面样式、交互逻辑、结果展示
- survey.json：题目数据、权重配置、战队画像、自适应加试题
- README.md：项目说明文档

## 运行方式

建议使用本地静态服务器打开，否则浏览器可能无法正确读取问卷数据。

### 方式一：使用 Python

在项目目录下运行：

Python -m http.server 8000

然后在浏览器中打开本地 8000 端口。

### 方式二：使用任意静态服务器工具

只要能正常提供静态文件访问即可。

## 判定逻辑说明

测试不是简单统计某个选项出现几次，而是综合了：

- 每道题的单独分值
- 不同题目的权重
- 战队画像匹配
- 接近结果时的自适应加试题
- 作答稳定度与结果置信度

这样可以更好地区分一些风格接近的队伍，例如：

- Vitality / NAVI / Astralis / PV
- FaZe / G2 / Falcons
- Spirit / FURIA / MongolZ

## 如何自定义

如果你想继续调整测试，只需要改 survey.json 中的内容：

- 修改 teams：调整战队描述和画像
- 修改 questions：调整题目、选项和分值
- 修改 weight：改变某道题的重要性
- 修改 tiebreakers：增加用于细分结果的加试题
- 修改 adaptive：控制何时触发加试

如果想改页面视觉或布局，则修改 index.html 即可。
